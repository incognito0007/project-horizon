# 🌅 Project Horizon

> _A structured 5-month self-development journey across modern web, backend, and systems programming._

---

## Overview

Project Horizon is a personal engineering growth initiative focused on deepening expertise across a modern full-stack skill set — building something meaningful every single day, and tracking that progress publicly through consistent commits.

This monorepo contains all daily practice code, experiments, mini-projects, and notes produced throughout the journey.

---

## Repository Structure

```
project-horizon/
├── web/          # React · Next.js · TypeScript · Storybook
├── dotnet/       # C# · ASP.NET · Entity Framework · xUnit
├── java/         # Java · Spring Boot · Spring Data JPA
└── notes/        # Daily logs · resources · reflections
```

### `/web`

Frontend and full-stack web work. Components built in isolation with Storybook, pages and routing in Next.js, everything strictly typed with TypeScript.

### `/dotnet`

C# experiments, ASP.NET APIs, Entity Framework projects, unit/integration tests, and design pattern implementations. Builds on existing production experience.

### `/java`

Java from the ground up — OOP, Collections, Streams, and a full Spring Boot REST API by the end of the journey.

### `/notes`

Daily learning logs in Markdown. One file per week. Honest reflections on what clicked and what didn't.

## Commit Convention

Every weekday has a commit. Format:

```
[web] Add useFetch custom hook with abort controller
[dotnet] Implement Repository pattern with EF Core
[java] Add Stream API exercises – filter/map/reduce
[notes] Week 03 log – TypeScript generics clicked today
```

## Guiding Principles

- **One commit a day, every weekday.** Consistency over intensity.
- **Build, don't just read.** Every concept gets a working code example.
- **Strict TypeScript always.** No `any`, no shortcuts.
- **Document as you go.** Future-me will thank present-me.
